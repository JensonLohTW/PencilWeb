# 網站規劃1

------

## 1) 【站點定位摘要】

- **目標受眾：**
  - B2B：中小企業（AI 應用導入、資料介接/推播、智慧空間）、教育訓練/航空相關單位、展演/體驗館與品牌展示
  - B2G/產業：智慧城市、工業 IoT、產業數據平台合作方（需【待補】）
  - 共同：正在找「可落地」的 VR/AR/MR/XR + AI 解決方案與系統整合團隊
- **核心價值主張（1 句）：**
  - **把 AI 與 XR 融入真實場域，打造可落地的沉浸式互動與智慧空間體驗。**
- **品牌關鍵字（5–8 個）：**
  - XR（VR/AR/MR）、沉浸式、智慧空間、AI Agent、資料介接、主動推播、五感互動、元宇宙 OASIS
- **主要轉化目標（CTA）：**
  - **預約 Demo / 洽談合作**
  - 下載方案簡報（【待補：是否提供】）
  - 填寫需求表單 / 聯絡我們

### Page 1：首頁 / `/**`

- **這頁的唯一目的（1 句）：**
  - 在 10 秒內說清楚你們做什麼、能交付什麼，並把人導向「解決方案」或「預約 Demo」。
- **版面結構（由上到下區塊列表）：**
  - **Section 1：Hero（品牌主張 + 立即轉化）**
    - 內容要點
      - Slogan：**「一切空間的動態都來自於 Pencil」**
      - 1 句價值主張（上面摘要）
      - 兩顆 CTA：預約 Demo（主）/ 看解決方案（次）
      - 補充：主打 XR + AI + 智慧空間（3 個關鍵詞）
    - 建議版式：全寬 Hero（左文右視覺或置中）
    - CTA：
      - Primary：**預約 Demo**（右上角固定按鈕 + Hero 內）
      - Secondary：**探索解決方案**
    - 素材需求：品牌視覺（3D/抽象粒子）、一句短副標、Logo（【待補】）
    - 動效/互動（可選）：首屏文字逐段進場、CTA 微彈跳、背景粒子視差
    - Three.js 建議（可選）
      - 場景概念：抽象「空間動態」線框/粒子流（像空間向量場）
      - 互動方式：滑鼠/陀螺儀輕微視差、滾動時粒子流方向變化
      - 風險/效能備註：LCP 風險 → **延後載入、首屏先用靜態 poster + fallback Canvas**
  - **Section 2：你們能解的問題（Pain → Solution）**
    - 內容要點
      - 教育訓練：沉浸式訓練提升理解與安全
      - 展示/導覽：AR 把資訊貼到現場
      - 空間智慧化：IoT + 互動 + 可視化
      - 企業 AI：資料查詢、AI chat、Agent 自動化
    - 建議版式：四欄卡片（含 icon + 一句）
    - CTA：每卡「了解更多」→ 導到 solutions 對應區塊
    - 素材需求：4 個 icon、每卡 1 句說法（可由你提供）
    - 動效/互動：卡片 hover 發光邊框、進場淡入
  - **Section 3：解決方案總覽（4 solutions）**
    - 內容要點：VR/MR、AR、智慧空間、AI for SME（每項 2–3 bullet）
    - 建議版式：左側選單 + 右側內容（Tab/Accordion）
    - CTA：每項「查看方案」/「詢問報價」
    - 素材需求：每方案示意圖或短影片（【待補】）
    - 動效/互動：Tab 切換內容滑入；手機改 Accordion
  - **Section 4：已完成專案（能力背書）**
    - 內容要點：你列出的 5 類專案（可做成「標題 + 交付內容」）
    - 建議版式：案例卡片 Grid（2–3 欄）
    - CTA：看更多案例 → /projects
    - 素材需求：每案 1 張圖（或 placeholder）、成果描述（【待補：可公開內容範圍】）
    - 動效/互動：卡片 hover 顯示「交付項目」
  - **Section 5：技術核心（OASIS 願景 + 核心堆疊）**
    - 內容要點
      - XR（VR/MR/XR）、全向跑步機、觸覺回饋、AI & 大數據、5G/6G、五感
      - 你們強項：系統整合 + 場域落地（建議明說）
    - 建議版式：時間軸/分層圖（Stack Diagram）
    - CTA：前往技術核心 → /technology
    - 素材需求：技術圖標、裝置照片（若可）
    - Three.js 建議（可選）
      - 場景概念：3D「技術堆疊塔」或「環形軌道節點」
      - 互動方式：滾動對應高亮節點（Scroll-driven）
      - 風險：3D 物件多 → **使用 instancing / 低面數 / 只在桌機啟用**
  - **Section 6：最後轉化區（Contact Strip）**
    - 內容要點：一句承諾 + 3 個你需要用戶提供的資訊（產業/目標/時程）
    - 建議版式：全寬 CTA Banner
    - CTA：預約 Demo / 填需求
    - 素材需求：背景圖或抽象漸層
- **SEO 建議：**
  - Title：空間動態科技 Pencil｜XR（VR/AR/MR）× AI × 智慧空間解決方案
  - Meta：提供 VR/MR 沉浸式訓練、AR 視覺化與智慧空間 IoT 整合，並協助中小企業導入 AI Agent / AI Chat，打造可落地互動體驗。
  - H1：XR × AI 打造可落地的空間互動體驗
  - 主要關鍵字：VR 教育訓練、AR 解決方案、智慧空間 IoT、AI agent、沉浸式體驗
- **RWD 注意事項：**
  - Hero 視覺改上圖下文；Tab 改 Accordion；案例卡片改單欄滑動；Three.js 桌機優先、手機 fallback 靜態背景。

## 4) 【共用設計與技術規範】

- **視覺風格（科技感）：**
  - 版面：12 欄網格、段落留白大、卡片圓角（12–16px）
  - 色彩：深色底（黑/深藍）+ 霓虹描邊/漸層光暈（避免過亮影響可讀性）
  - 元件：玻璃擬態（低透明 + 模糊）用在卡片/浮層；關鍵 CTA 用高對比
  - 背景：微粒子/噪點紋理、柔和光束、極淡網格線（提升科技感但不搶字）
  - 字體：中英混排可用 Inter/Manrope + 思源黑體（【待補：品牌字體偏好】）
- **動效策略：**
  - 首屏：文字分段進場（50–150ms stagger），背景微動（視差）
  - 滾動：Scroll-driven 高亮（用 IntersectionObserver），避免整頁 heavy animation
  - Hover：卡片邊框發光、icon 輕旋/位移（小而精準）
  - 頁面切換：淡入淡出 + 內容位移（80–200ms），避免大位移造成暈眩
- **Three.js 使用原則：**
  - **何處用：**首頁 Hero 抽象背景、Technology 節點網路（資訊視覺化）
  - **何處不用：**表單頁、長文資源頁、內容密集的比較表（以可讀性/效能為先）
  - **Fallback：**同一視覺準備靜態 poster（webp）+ CSS/SVG 動效版本
  - **Lazy load：**進入 viewport 再載；prefers-reduced-motion 直接停用
  - **LCP 影響：**首屏先渲染文字與 CTA，Three.js 延後（requestIdleCallback / setTimeout）
- **效能/SEO：**
  - 圖片策略：全站 WebP/AVIF；首屏圖 `preload`；其餘 `lazy`
  - 資源拆分：Three.js 獨立 chunk、按頁載入；影片用 poster + click-to-play
  - SSR/CSR 建議：官網建議 SSR/SSG（Next.js）提升 SEO；互動區塊 CSR island
  - 結構化資料（如適用）：
    - Organization、WebSite、BreadcrumbList
    - 有案例可用 CreativeWork/Project（【待補：可公開案例細節】）
